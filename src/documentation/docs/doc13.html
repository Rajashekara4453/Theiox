    Notifications

    Its an indication of an undesirable condition detected by your system such as a device that is not responding to input commands. The alarm alerts operators that this condition needs to be fixed.

    Accessing Alert Notifications

    • Click on Alert Notification -> ** Configure Alert Notification**

    Configure Alert Notifications

    • Click Alert Notifications and click +Create

    • You have following options for configuration

      • Under General Info

      • Name : Alarm Display Name

      • Discription : Breif Discription about Alarm

      • Asset : The Device for which the alarms are set.

      • Type : Select if its **Alarm or Events88

      • Trigger on : Here you can select when would like to trigger Alaram like Day, *Hrs ect..

      • Acknowledgement Required Checkbox : Acknowledging an alarm lets other users know that you are taking ownership of the issue. After an alarm is acknowledged, its alarm actions are discontinued. The user acknowledges the triggered alarm to let other users know someone is working on the problem and to prevent the alarm from sending more email messages.

      • Priority : Each alarm has a priority level. Use the priorities to distinguish between events that require immediate action and those that do not require action.

      • Under Rule Set Under Rule Set Select Tag, apply Condition and, Set Value click save and proceed.

      • Tag

      • Condition

      • Set Value

      • Under Action Under Action you can add ** Notification Profile** this is configure the users in your organisation who should receive the Alarms & Event notification and the mode of delievery.

      • User/UserGroup

      • Notification Type

    Edit Alarm/Event

    To Edit the Alarm & Event configuration

    • Click on Configure Alert Notifications and click Pencil Icon, make the changes and click Save Alram.

    Alert Notification List

    will display following

    Asset: Shows the device Name Alert Name : Name of the Alert
    Type: Show wheather it is Alarm/ Status Priority : Shows the priority type like High, low,
    Status Acknowledge : Shows Aknowledgement Status of Alarm
    Tag name : Show the parameter set for the Alarnm
    Asset Status : Shows weather device is in ON/OFF status Set Value : Shows what was the value set
    Triggered value: Shows what was the value set to trigger the Alarm Triggered Time : Shows the time when Alarm was Triggred Reset Time : Shows the time when was the Alarm reset

    • Cards of top dispalys the count of the Alram and the its types

    Priority Type

    • Priority Types gives you the option to create your own priority
      • Click + add a new Priority type for the Alarms.
      • Enter the Alarm Priority Name, a brief description, Priority Level and Priority color. After entering all the fields, click on save button.

    Edit Priority Type

    • To Edit/update click on the list available on the left or search the Alarm Priority type in the Search box
    • Once you have edited click on the Update button.
    • If you want to delete the Alarm Priority type click on the Delete button.
